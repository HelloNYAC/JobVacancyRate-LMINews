LIMITATIONS

API Data Only Available at the Provincial Level

→ Originally, I wanted to find out (1) Job Vacancy Rate for different NOCs; (2) Specific NOC related LMI News; (3) Job Vacancy Rate and LMI News for certain Economic Region. But Statistics Canada API only provides data at the provincial level.

→ Although our team has access to Job Vacancy Rate for each Economic Region but my goal was to practice using APIS. I decided to stick with whatever the API provides.

→ The job vacancy and job vacancy rates at the provincial level are still useful information to a decision maker.

Limited API methods from Statistics Canada

→ Statistics Canada API only provides limited methods, gaining direct access to data was a challenge.

→ Lots of preparation work needed in order to use Statistics Canada API methods.

→ GoogleSheet API was another tool I used during the preparation work.

Data Analysis Skills

→ The longest time I spent on this project was to understand the excel data table from Statistics Canada and understand the Coordinate IDs and Vector IDs because those are the only methods available to access to the data.

→ I didn't understand the rule of nameing these IDs until I got stuck with different API methods. I then realized the key is to know the rule of naming the IDs.

→ I all made sense when I understood the rule.

A Few DAM Methods Were Being Updated

→ A few methods in the DAM were in the middle of being updated, so I was not able to benefit from accessing a custom table through an API for this project.

→ I would like to revisit this project in the future using the DAM when these methods are updated.
